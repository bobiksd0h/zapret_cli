# zapret_cli

Удобный терминальный интерфейс (TUI) для установки и управления **запретом** на Linux.
Под капотом — движок [bol-van/zapret2](https://github.com/bol-van/zapret2) (nfqws2,
Lua-десинхронизация); проект — порт [zapret.installer](https://github.com/Snowy-Fluffy/zapret.installer) / [zapret.cfgs](https://github.com/Snowy-Fluffy/zapret.cfgs) (которые были
сделаны под первую версию запрета) на этот движок.

## Установка

Из корня репозитория:

```sh
sudo sh installer.sh
```

Дальше меню вызывается командой:

```sh
sudo zapret
```

Установщик ставит зависимости, скачивает движок в `/opt/zapret2`, применяет стратегию
и хостлисты из [`cfgs/`](cfgs), создаёт команду `zapret`. Стратегии берутся локально из
`cfgs/` (в сеть ходим только за бинарниками движка).

## Структура

```
installer.sh         bootstrap: деплой в /opt/zapret.installer и запуск меню
zapret-control.sh    точка входа: подключает модули, поднимает права, рисует меню
files/               модули установщика (vars, utils, init, service, config, menu, install)
cfgs/                стратегии, хостлисты, fake-пейлоады  (см. cfgs/README.md)
```

## Возможности меню

- статус сервиса/автозагрузки, запуск/остановка/перезапуск;
- смена стратегии (вручную или автоподбор по доступности доменов);
- смена хостлиста, добавление/удаление/поиск доменов и IP (вкл. exclude-лист);
- игровой режим (десинхронизация всего трафика на игровые порты);
- прямое редактирование конфигов, кастомные стратегии/листы;
- обновление движка с сохранением конфигурации и хостлистов;
- удаление.

## Стратегии

Перенесены **все стратегии** первой версии (54 файла в `cfgs/configurations/`).
В новом движке стратегии задаются не C-опциями `--dpi-desync-*`, а профилями с вызовами
Lua-функций десинхронизации (`--lua-desync=fake:blob=…`, `--lua-desync=multisplit:pos=…`,
`--lua-desync=hostfakesplit:host=…` и т.д.). Полная таблица соответствия и детали — в
[cfgs/README.md](cfgs/README.md).

ip-листы оставлены без изменений — формат хостлистов не поменялся.

> Примечание: движок устанавливается в `/opt/zapret2`, а его системный сервис называется
> `zapret2` — эти внутренние имена фиксируются апстрим-установщиком и не переименовываются.
> В меню и команде используется просто «запрет» / `zapret`.
